A round of fixes from live testing — hover feedback, avatars surviving a storage reset, and a clearer heads-up when something in Settings needs your attention.

## Fixes

- 🖱️ **The games-list refresh button actually reacts to hover now** — same root cause as a previous fix (an inline style beating the CSS rule's specificity), just on a different button this time.
- 🖼️ **Your avatar survives deleting and recreating your storage** — avatars live inside the shared repo itself, so a fresh (or recreated) storage started out with none. Creating a storage, or connecting to a friend's, now pushes your local avatar right away if you already have one set.
- 🔔 **"Settings" gets a colored dot when it actually matters** — red if you lost access to a storage (kicked, or it got deleted), amber for a declined/skipped sync event — instead of only the plain "update available" dot it had before.
- 🏷️ **Renamed a confusing button** — the "exclude files from sync" card reused the generic "Try again" label on a button that just re-scans the save folder, no error involved. Now says "Refresh file list".
