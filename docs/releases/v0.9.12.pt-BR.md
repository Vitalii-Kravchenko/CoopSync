Uma correção de perda de dados — restaurar um save enquanto o jogo ainda não tinha fechado de verdade podia apagar silenciosamente a sessão que você acabou de jogar.

## Correções

- 🛡️ **Restaurar agora verifica se o jogo está realmente fechado** — restaurar uma versão antiga do save enquanto o processo do jogo ainda estava rodando (mesmo que só demorando um pouco pra fechar depois de sair) sobrescrevia os saves locais recém-jogados com a versão antiga. Como Restaurar também envia o resultado imediatamente, o autopush em segundo plano que rodava logo depois não encontrava mais nada para enviar — a sessão se perdia silenciosamente para sempre. Agora Restaurar recusa a ação com uma mensagem clara se o jogo ainda não fechou de verdade.
