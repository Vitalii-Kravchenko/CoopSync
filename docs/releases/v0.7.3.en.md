A small but important patch — one of the fixes here resolves a bug that could permanently break sync for a co-op partner.

## Fixes

- 🔧 **Critical: sync could break permanently.** If the app closed or crashed mid-upload, its internal working copy could be left in a "dirty" state — and every sync afterward (history, upload, download) would fail with a "local changes would be overwritten by merge" error, with no way to recover except manual cleanup. The app now automatically resets that internal state before syncing, so this can no longer happen.
- 👤 In the members list (when you've joined a friend's storage), the "owner" row used to show your own nickname instead of the actual storage owner — now it always shows the real owner.
- 🧭 Fixed the "no storage access" warning on the connect screen being misaligned to the left instead of lining up with the rest of the setup steps.
