# CoopSync v0.4.0

**English** · [Français](v0.4.0.fr.md) · [Deutsch](v0.4.0.de.md) · [Polski](v0.4.0.pl.md) · [Português (Brasil)](v0.4.0.pt-BR.md) · [Русский](v0.4.0.ru.md) · [简体中文](v0.4.0.zh-CN.md) · [Español](v0.4.0.es.md) · [Türkçe](v0.4.0.tr.md) · [Українська](v0.4.0.uk.md)

## New

- 👥 **Friends tab** — friend management moved out of Settings into its own sidebar screen. Invite by GitHub username and see who's accepted, who's still pending.
- 🗑️ **Delete repository** — a guarded danger-zone action in Settings to permanently delete the shared saves storage: a confirmation dialog with a 10-second countdown before the delete button unlocks.

## Fixes

- 🐛 Fixed a save-sync bug: quitting a game always pushed your local save to GitHub, even if a friend had already pushed a newer version while you were playing together — silently overwriting their progress. CoopSync now checks the cloud version first and skips the push (with a warning banner) if it's already ahead.
- 🐛 Failed background sync events used to fail silently with no feedback — now shown as a banner too.

## Other changes

- 📘 README: replaced hand-styled badges with real shields.io badges (GitHub strips inline `style` attributes from README HTML, so the old ones rendered as plain text), a clearer requirements table, and an explicit note that Git for Windows is the only thing you install yourself.
