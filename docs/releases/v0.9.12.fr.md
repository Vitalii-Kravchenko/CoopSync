Une correction de perte de données — restaurer une sauvegarde alors que le jeu n'avait pas encore complètement quitté pouvait effacer silencieusement la session que vous veniez de jouer.

## Corrections

- 🛡️ **La restauration vérifie maintenant que le jeu est vraiment fermé** — restaurer une ancienne version de sauvegarde pendant que le processus du jeu tournait encore (même juste un instant après avoir quitté) écrasait vos sauvegardes locales fraîchement jouées avec l'ancienne version. Comme la restauration envoie aussi le résultat immédiatement, l'auto-push en arrière-plan qui s'exécutait un instant plus tard ne trouvait plus rien à envoyer — la session était silencieusement perdue pour de bon. La restauration refuse désormais l'opération avec un message clair si le jeu n'a pas encore complètement quitté.
