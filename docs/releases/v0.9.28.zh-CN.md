这次终于找到并实测验证了自定义游戏封面无法同步给队友的真正根源——另外，新增游戏和删除操作现在也能保持同步了。

## 修复

- 🖼️ **找到了自定义游戏封面双方都无法同步的真正原因。** 自定义游戏的内部 ID 包含一个 `:` 字符，而在 Windows 上这并不是文件名中的非法字符——它会被悄悄当作隐藏数据流的标记来处理。封面每次都"保存成功"（从未报错），而真正的图片数据却悄悄消失在 git 看不到的文件系统角落里，所以从来没有真正进入过提交记录。修复了文件命名方式，清理了共享历史记录里那个损坏的残留文件，并实测确认新上传的封面现在能正确落地并同步给队友。
- 🤝 **刚添加的自定义游戏不再悄悄同步失败。** 把一个全新的自定义游戏注册给队友，是和本地保存分开的一个步骤，之前可能悄悄失败（比如和队友后台同步时机撞在一起），却没有任何提示告诉你出了问题。现在会在后台自动重试，直到成功为止。
- 🗑️ **删除自定义游戏现在真的会把它也从队友那边删除。** 之前删除操作只是在你自己电脑上停止追踪这个游戏——已经见过这个游戏的队友会永远保留它。现在队友那边的副本会在几分钟内自动消失，他不需要做任何操作。
- ⏳ **刚添加的游戏不会再永远卡在"检查中…"了。** 添加游戏会刷新游戏列表，但不会刷新它的同步状态，所以它就一直悬而未决，直到某个不相关的操作碰巧刷新了状态。
