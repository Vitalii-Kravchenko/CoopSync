一个可靠性补丁 —— 最重要的部分修复了两个可能永久破坏队友同步功能的 bug。此外还新增了:头像共享。

## 新增功能

- 🖼️ **头像现在会在你和朋友之间同步了。** 在设置中设置自定义头像 —— 它会上传到共享存档库,你的队友也能看到(你也能看到他的),不再是普通的占位图标。

## 修复内容

- 🔧 **严重问题:同步可能永久损坏。** 如果应用在上传存档过程中被关闭或崩溃,内部工作副本可能会残留在"脏"状态 —— 此后任何同步操作(历史记录、上传、下载)都会因 "local changes would be overwritten by merge" 错误而失败,且无法自行恢复,只能手动清理。现在应用会在每次同步前自动重置该内部状态。
- 🔑 **严重问题:上传可能完全失败**,报错 "unable to auto-detect email address",出现在从未配置过用户名/邮箱的 git 环境中。现在应用会自行设置提交身份,不再依赖机器本地的 git 配置。
- 👤 同步历史和"最后更新者"信息现在能正确显示实际执行操作的人 —— 之前总是显示存档库所有者,即使实际同步的是你的队友。
- 👤 在成员列表中(当你加入了朋友的存档库时),"所有者"一行原本显示的是你自己的昵称,而不是真正的所有者 —— 现在始终显示正确的人。默认头像(尚无图片时)现在在各处也保持一致。
- 🔄 好友标签页不再显示过期数据 —— 每次打开该标签页时都会自动刷新,而不是只在应用启动时加载一次。
- 🪟 再次启动应用(例如在应用已在托盘运行时双击快捷方式)不会再打开重复的实例 —— 只会将已打开的窗口置于前台。
- 🧭 修复了连接界面上"无法访问存档库"警告文字的错位问题 —— 之前它偏向左侧,而不是与其他设置步骤对齐。
