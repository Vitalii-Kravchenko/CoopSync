Rejoindre le stockage d'un ami ne nécessite plus un aller-retour séparé sur GitHub pour accepter quoi que ce soit.

## Corrections

- 🤝 **Rejoindre accepte désormais automatiquement une invitation en attente** — une invitation de collaborateur GitHub doit être explicitement acceptée avant que l'accès ne prenne vraiment effet. « Se connecter à un ami » se contentait auparavant de vérifier l'accès et affichait « pas d'accès — demande à ton ami de t'inviter », même juste après qu'il l'ait fait, et redémarrer l'application n'y changeait rien puisque ce n'était pas un problème de cache — GitHub lui-même n'avait tout simplement pas encore accordé l'accès. Il vérifie maintenant s'il existe une invitation en attente vers le stockage de l'hôte et l'accepte automatiquement, donc rejoindre fonctionne dès que l'hôte a envoyé l'invitation.
