A single fix for a duplicate autostart entry some installs had picked up over time.

## Fixes

- 🚀 **CoopSync no longer starts twice with Windows.** An older version (before the app had an explicit identity for Windows) registered its autostart entry under a default, auto-generated name. A later version started managing autostart under the app's real identity instead, but never cleaned up that old entry — so both fired on every login, launching a redundant second process (immediately closed, but still visible as an extra entry in Windows' startup apps list and Task Manager). This update removes the stale entry automatically on first launch.
