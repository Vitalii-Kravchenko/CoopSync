Es gibt hier keinen Server — die App weiß nur Bescheid, wenn sie GitHub erneut fragt. Zwei Stellen, an denen sie nicht oft genug gefragt hat.

## Fehlerbehebungen

- 🔄 **Der Tab "Freunde" bemerkt eine angenommene Einladung von selbst** — jemanden einzuladen ließ bisher "wartet auf Annahme" auf dem Bildschirm hängen, selbst nachdem die Person tatsächlich angenommen hatte, bis man zu einem anderen Tab wechselte und zurück. Er fragt jetzt alle 5s nach, solange eine Einladung aussteht und man den Tab tatsächlich ansieht, und hört von selbst auf, sobald niemand mehr aussteht.
- 🔄 **Einstellungen bemerken, wenn ein Host dich entfernt** — anders als die anderen Tabs prüften die Einstellungen den Speicher-/Host-Status nach dem ersten Laden nie erneut, sodass ein entfernter Mitarbeiter unbegrenzt weiter "verbunden mit dem Speicher von X" sah. Sie lesen jetzt beim Zurückkehren zum Tab neu ein, und auch in dem Moment, in dem die Benachrichtigung über entzogenen Zugriff im Hintergrund eintrifft. Das deckte gleichzeitig einen zweiten Fehler auf: Sobald der Status korrekt "kein Zugriff" anzeigte, bot der Leerzustand eine für Hosts gedachte Schaltfläche "Speicher erstellen" an — ein Klick darauf hätte still einen verwaisten Speicher unter dem eigenen Konto erstellt, während weiterhin auf den alten Host verwiesen wurde. Ersetzt durch eine klare Meldung und eine Schaltfläche in den bestehenden Verlassen/Eigene-Kopie-behalten-Ablauf.
