A reliability patch focused on error handling — no new features, just clearer messages when something goes wrong, and a few spots that used to hang or show a misleading status now don't.

## Fixes

- 🌐 **Clear "no internet connection" messages for GitHub actions** (signing in, setting up storage, inviting a friend) — some of these used to show a raw technical error instead.
- ⏳ The app no longer hangs forever on "Loading…" if a network check fails right at startup — it now shows a clear error with a "Try again" button.
- 🔑 A temporary network hiccup while checking your session no longer signs you out — only a genuinely expired session does that now.
- 📦 Settings, Friends, and the setup screen no longer falsely claim storage isn't set up when a network check simply failed — they show the real error with a retry option instead.
- 🎮 The games list no longer hangs forever on "Looking for games…" if something goes wrong loading it.
- 🔄 Auto-sync no longer silently stops working if it can't check which games are running — you'll see a message instead of it quietly going dark.
- 📡 Signing in now survives a brief network blip instead of aborting the whole login.
- ⏱️ New message when GitHub's API request limit is temporarily reached, telling you when to try again.
