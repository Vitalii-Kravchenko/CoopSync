The actual root cause of custom game covers not reaching a partner, found and live-tested this time — plus new games and removals now stay in sync too.

## Fixes

- 🖼️ **Found the real reason a custom game's cover never reached a partner, on either side.** A custom game's internal ID contains a `:` character, and on Windows that's not an invalid filename character — it's silently treated as a hidden-data-stream marker. The cover was "saving" successfully every time (no error, ever) while the actual image data quietly vanished into a part of the file system git can't see, so it never made it into a real commit. Fixed the file naming, cleaned up the broken leftover in the shared history, and confirmed live that a fresh upload now lands correctly and reaches a partner.
- 🤝 **A custom game you just added no longer silently fails to reach a partner.** Registering a brand-new custom game with a partner is a separate step from saving it locally, and it could fail quietly (e.g. a timing clash with a partner's own background sync) with nothing telling you it happened. Now retried automatically in the background until it succeeds.
- 🗑️ **Removing a custom game now actually removes it for your partner too.** Previously, removal only stopped tracking it on your own PC — a partner who'd already seen the game kept it forever. Now their copy disappears within a couple of minutes, no action needed on their end.
- ⏳ **A freshly-added game no longer gets stuck on "Checking…" forever.** Adding a game refreshed the games list but not its sync status, so it just sat unresolved until something unrelated happened to refresh it.
